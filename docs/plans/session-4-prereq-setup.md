# Session 4 Prerequisite Setup

## Goal

Session 4 of `security-hardening-v2-phase-2-implementation.md` requires a
clean local install with the current Phase 1 credential contract in `.env` and
all infrastructure pods running with the per-service credentials generated by
Tilt.

## Required Variables

The following values must exist in `.env` alongside the existing Auth0 and FRED
credentials:

- `POSTGRES_BOOTSTRAP_PASSWORD`
- `POSTGRES_TRANSACTION_SERVICE_PASSWORD`
- `POSTGRES_CURRENCY_SERVICE_PASSWORD`
- `POSTGRES_PERMISSION_SERVICE_PASSWORD`
- `RABBITMQ_BOOTSTRAP_PASSWORD`
- `RABBITMQ_CURRENCY_SERVICE_PASSWORD`
- `REDIS_DEFAULT_PASSWORD`
- `REDIS_OPS_PASSWORD`
- `REDIS_SESSION_GATEWAY_PASSWORD`
- `REDIS_EXT_AUTHZ_PASSWORD`
- `REDIS_CURRENCY_SERVICE_PASSWORD`

Use the current values from `.env.example` as the local-development baseline.

## Procedure

```bash
# 1. Ensure .env contains the current Infrastructure Credential Contract
#    from .env.example plus your Auth0 and FRED values.

# 2. Bootstrap the local environment
./setup.sh

# 3. Start everything
tilt up

# 4. Once all pods are healthy (check Tilt UI or kubectl get pods -A), verify
./scripts/dev/verify-security-prereqs.sh
./scripts/dev/verify-phase-1-credentials.sh
```

If both verification scripts pass, Session 4 prerequisites are met.
